# Copilot Skill Generator

A tool that auto-generates GitHub Copilot skills for the Infineon Design System.

## What it generates

Two Copilot skills:

1. **`infineon-design-system`** - Component documentation with props, events, CSS properties, and usage guidance
2. **`infineon-design-uiux`** - UI/UX design guidance

## Generated Output Structure

```
.github/skills/
├── infineon-design-system/
│   ├── SKILL.md              # Main skill file
│   └── references/
│       ├── components/       # 47 component docs
│       ├── foundations/       # 9 design foundation docs (tokens only)
│       └── guides/           # Getting started, theming, icons, etc.
│
└── infineon-design-uiux/
    ├── SKILL.md
    └── references/
        └── design-guidance/   # 9 UI/UX guidance docs
```

## Prerequisites

- Node.js 18+
- npm or pnpm

## Quick Start

```bash
# Navigate to the generator directory
cd scripts/copilot-skill-generator

# Install dependencies
npm install

# Generate both skills
npm run generate        # infineon-design-system
npm run generate:design # infineon-design-uiux
```

## Commands

| Command | Description |
|---------|-------------|
| `npm run generate` | Generate `infineon-design-system` skill |
| `npm run generate:design` | Generate `infineon-design-uiux` skill |

## Configuration

### Main Skill (`config.json`)

Controls what gets included in `infineon-design-system`:

```json
{
  "components": {
    "sourceDir": "packages/components/src/components",
    "outputDir": ".github/skills/infineon-design-system/references/components"
  },
  "foundations": {
    "sourceDir": "packages/storybook/stories/foundations",
    "outputDir": ".github/skills/infineon-design-system/references/foundations"
  }
}
```

### Design Skill (`config-design.json`)

Controls what gets included in `infineon-design-uiux`:

```json
{
  "design": {
    "outputDir": ".github/skills/infineon-design-uiux/references/design-guidance"
  }
}
```

## How It Works

### Data Sources

The generator parses these source files:

| Source File | What it provides |
|-------------|------------------|
| `*/readme.md` | Props and Events tables (auto-generated by Stencil) |
| `*/Usage.mdx` | Usage guidance, anatomy, variants (Storybook docs) |
| `*.tsx` | CSS custom properties, Shadow DOM parts |
| `packages/storybook/stories/foundations/*.mdx` | Design foundation tokens |

### Parser Modules

- `componentParser.js` - Parses component readme.md and Usage.mdx files
- `foundationParser.js` - Parses design foundation MDX files, extracts tokens
- `tokenParser.js` - Extracts design tokens from `@infineon/design-system-tokens`
- `iconParser.js` - Extracts icon list from `@infineon/infineon-icons`
- `usageParser.js` - Parses framework installation commands from root USAGE.md

### Generator Modules

- `skillGenerator.js` - Creates SKILL.md with version tracking
- `componentGenerator.js` - Generates component documentation
- `guideGenerator.js` - Generates guides (theming, icons, troubleshooting)
- `designGuidanceGenerator.js` - Generates UI/UX design guidance

## Template System

Uses Handlebars templates:

- `component.hbs` - Component documentation template
- `SKILL.md.hbs` - Main skill file template

## What Gets Included

### Component Docs Include:
- Props (from readme.md table)
- Events (from readme.md table)
- CSS Custom Properties (extracted from TSX `setProperty` calls)
- Shadow DOM Parts (from `@part` annotations in TSX)
- Usage content (from Usage.mdx - anatomy, variants, when to use)

### Foundation Docs Include:
- Title
- Tokens table (extracted from MDX tables)

### Excluded (not useful for Copilot):
- Installation boilerplate (now in SKILL.md/guides)
- Framework-specific code examples (now in guides)
- Empty sections (no props/events = no section)
- Generic placeholder text ("See the component index...")
- Raw MDX components and imports

## Version Tracking

Version is automatically detected from:
```
packages/components/package.json -> version
```

And embedded in generated SKILL.md files.

## Troubleshooting

### Empty props/events for a component

Check if:
1. The component has a `readme.md` file
2. The readme.md has properly formatted tables
3. The component isn't a container directory with subcomponents (e.g., `navigation/`, `select/`)

### HTML entities showing (e.g., `&quot;` instead of `"`)

This is handled automatically. The template uses triple braces `{{{variable}}}` to prevent Handlebars HTML-escaping.

### Icons/Tokens lists are too long

This is intentional. The exhaustive lists help Copilot avoid hallucinating wrong icon/token names.

## Extending

### Adding a new parser

1. Create parser in `src/parsers/`
2. Export an async function that returns structured data
3. Import and call in `src/index.js`

### Modifying component template

Edit `src/templates/component.hbs`. Available data:

```handlebars
{{name}}           # Component name (e.g., "button")
{{displayName}}    # Display name (e.g., "Button")
{{description}}    # From readme.md
{{usageContent}}   # Parsed from Usage.mdx
{{props}}          # Array of {name, type, default, description}
{{events}}         # Array of {name, detail, description}
{{cssProperties}}  # Array of {name, cssVar, default}
{{cssParts}}       # Array of {name, description}
```
